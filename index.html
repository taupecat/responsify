Introduction

Who Started It?

Ethan Marcotte in A List Apart (2010)

Built upon John Allsop's "The Dao of Web Design"
too often, web designers had a fixed-canvas mindset, and didn't consider the fuidity of the web

Why Does It Exist?

  • Reaction to mobile vs. desktop sites
  • Lines are blurred by a multitude of devices
  • Increasingly, all around the world, a mobile device is a person's primary means of accessing the Internet
  • Multiple code bases make sites harder, more expensive to maintain
  • "Server Attention Span" problem
  • Browser-sniffing problem

Devices shown are just the latest on the market. Consider older devices, future devices

What Is It?

  • Fluid layouts
  • Flexible images and media
  • CSS3 media queries

Images

Battle between "picture" element and "srcset" attribute

<picture> Element


<picture>
  <source srcset="high-res.jpg" media="(min-width: 1000px)">
  <source srcset="medium-res.jpg" media="(min-width: 400px)">
  <source srcset="low-res.jpg">
  <img srcset="fallback.jpg" alt="Our fallback image">
</picture>
  • Works much like <video> and <audio>
  • Uses built-in media queries
  • Includes an <img> fallback
  • Coming soon to a browser near you
  • Picturefill.js enables support of the <picture> element
  • add_image_size()
Using WordPress' add_image_size() function to auto-scale and crop your images works nicely with a picturefill solution. There's even a plugin for doing that.

demo

Responsive JPEGs

  • Save as "progressive"
  • High compression, 2.2x size
Don't do the high compression/double size thing in sliders. Heck, don't use sliders.

demo

Warning for Responsive Images in WordPress

  • WordPress uses "width" and "height" attributes
  • Use "width" and "max-width" CSS properties to override
  • Always set "height: auto;" or your images will be distorted

Layout

Box-Sizing

Know this, love this:


*, *:before, *:after {
  box-sizing: border-box;
}

Works in IE8+

demo

Flexbox


.container {
  display: flex;
}
  • Several older implementations exist in the wild
  • IE10+ support (but with older syntax)
This is really neat, but not yet widely understood. IE support limits usefulness now, but modernizr can be used to detect support and provide fallbacks
